API Reference

Last updated: July 24, 2025 10 min read

The Mnemo API is a tool that creates a path between application and code. As a developer, you can use the API to get, set and create functionallity within the application.

Basics

The API follows a simple pattern, you can get, set, modify and create within the application.

The base of the API is this command: mnemo.api.category.function()

The categories are (with examples):

  • content - E.g: get(), set(), modify(), create()
  • ui - E.g: showModal(), toast()
  • system - E.g: readFile(), writeFile()
  • events - E.g: subscribe(), unsubscribe(), OnPathCreated(), onFileChanged()
  • ai - E.g: generatePath(), prompt(), summarize(), translate()
  • settings - E.g: get("Language"), set("Language", "en"), set("theme", "dawn")
  • extension - E.g: getID(), getPath(), openStorage()
  • tools - E.g: log(), formatDate(), sanitize(), getVersion()

A complete set of functions can be found in the API Reference

A few examples

The api can be used in a few different ways, here are a few examples:

  • string current_theme = mnemo.api.settings.getTheme()
  • bool is_key_down = mnemo.api.get.keydown("F1")
  • string keydown = mnemo.api.get.keydown()
  • mnemo.api.ui.createOverlay("myOverlayView.axaml","MyOverlayViewModel.cs","MyOverlayView.axaml.cs")
  • mnemo.api.ai.generatePath("First Aid", "Generate a path for first aid training")

Notification Service

Key Methods

  • mnemo.api.ui.toast.create("Title", "Message", "type") - Send a notification
  • mnemo.api.ui.toast.remove("Title") - Remove a notification
  • mnemo.api.ui.toast.removeAll() - Remove all notifications
  • mnemo.api.ui.toast.removeAllOfType("type") - Remove all notifications of a specific type

Data Service

Key Methods

  • mnemo.api.content.get.flashcards("deckId") - Retrieve flashcard data
  • mnemo.api.content.get.notes("subjectId") - Access study notes
  • mnemo.api.content.get.quizzes("quizId") - Retrieve quiz data
  • mnemo.api.content.get.mindmaps("mindmapId") - Access mind map data

Theme Service

Key Methods

  • mnemo.api.settings.set("theme", "dark") - Change the application theme
  • mnemo.api.settings.get("theme") - Get current theme colors

Event System

mnemo uses an event-driven architecture. Subscribe to events to respond to user actions and system changes:

Common Events

  • mnemo.api.events.subscribe("FlashcardDeckCreated", OnFlashcardDeckCreated); - Fired when a flashcard deck is created
  • mnemo.api.events.subscribe("MindmapCreated", OnMindmapCreated); - Fired when a mindmap is created
  • mnemo.api.events.subscribe("ThemeChangedEvent", OnThemeChanged); - Fired when the application theme changes
  • mnemo.api.events.subscribe("FlashcardDeckDeleted", OnFlashcardDeckDeleted); - Fired when a flashcard deck is deleted

Best Practices

Follow these guidelines when using the API:

  • Handle exceptions appropriately
  • Dispose of resources properly
  • Use async/await for asynchronous operations
  • Follow the MVVM pattern
  • Respect user privacy and data security

API Versioning

The mnemo API follows semantic versioning:

  • Major version changes indicate breaking changes
  • Minor version changes add new features
  • Patch version changes fix bugs
Note: Differences between the API version in the application and whats being created can lead to unexpected behavior, so always use a compatible version of the API.
