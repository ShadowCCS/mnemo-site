Making Games

Last updated: July 28, 2025 5 min read

Introduction

Games are a way to extend the functionallity of the application. The games themselves are made using MonoGame, which is a open-source, cross-platform game engine.

Mono Game is a really capable, it supports both 2D and 3D games and have been used to make games such as Stardew Valley. If you stretch it a bit, you can say Mono Game is the C# alternative to PyGame.

The reason we use Mono Game is that it uses the .NET framework, same as the core application, meaning we get to use the same framework for the entire application.

How Does The Game System Work?

Games are as mentioned made with Mono Game, then the game itself is embedded into the application. They are loaded dynamically at runtime, and similar to extensions, they are isolated from the core. The games access data from the core using the Mnemo API, which you could read about here.
